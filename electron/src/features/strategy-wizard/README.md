# strategy-wizard-v2

用途：智能策略模块平行重构目录（V2），用于与 `strategy-wizard` 并行开发与验证。

## 原则

1. V2 与现网目录并行，不直接改动旧模块行为。  
2. 所有新逻辑先在 V2 验证通过，再逐步切流。  
3. 旧模块仅接收必要修复，不继续堆叠复杂功能。  

## 目录结构

1. `components/`：V2 组件与页面容器  
2. `store/`：V2 单一状态源（Working/Saved/Active）  
3. `services/`：V2 接口适配层与数据仓库  
4. `utils/`：V2 工具函数  
5. `factors/`、`templates/`：可复用资产（按需迁移）  

## 重构阶段（与计划书对齐）

1. 阶段A：垫层（统一模型与接口适配）  
2. 阶段B：Step1/Step2 收敛（单一真相源）  
3. 阶段C：性能韧性优化（大池加载与失败兜底）  
4. 阶段D：清理旧模块并切主路由  

## 切换策略

1. 默认入口仍指向旧目录：`strategy-wizard`。  
2. 可通过环境变量 `VITE_USE_STRATEGY_WIZARD_V2=1` 将主入口 `/strategy-wizard` 指向 V2。  
3. V2 验收完成后，替换旧入口并移除旧目录。  

## 当前状态

1. 已创建骨架目录。  
2. 已接入独立路由：`/strategy-wizard-v2`。  
3. 已实现 Step1/Step2 最小闭环（DSL 生成工作池、CSV 导入覆盖工作池、保存/激活/复用历史池）。  
4. Step1 与 Step2 已共享同一 `workingPool` 状态，作为阶段B第一版验证基础。  
5. Step1 已补齐 V2 侧“资产库管理”与“我的云端股票池”视图，支持全市场加载、云端复用、同步状态展示，便于与旧模块对照验收。
6. 已完成阶段C第一版：全市场加载采用“两阶段显示（先代码后特征）”，并加入一次重试与错误提示去重，降低网关短时抖动对体验的影响。
7. 已完成 Step3 门禁：仅当 `activePoolVersionId` 已设置且 `dirty=false` 时允许进入后续流程，防止未保存状态误进入下一步。
8. 为保证视觉一致性与可用性，`/strategy-wizard-v2` 当前已切换为复用 `strategy-wizard` 原页面组件（同构 UI 模式），前端设计与交互保持不变，仅用于并行接入与验证。
