## 说明
- 归属路径：electron\src\components\modules
- 修改本目录代码后请同步更新本 README

## 交易记录模块说明（2026-02 更新）
- `TradeRecordsCard` 升级为固定 10 行交易表格，数据不足时渲染占位行，避免卡片高度和结构抖动。
- 行背景采用浅蓝到白色渐变，强化交易信息层次，保持低干扰阅读体验。
- 列表默认按时间倒序展示（最新在最上）。
- 交易记录表头已改为可见列标题，并统一为 5 列等分布局，保证时间、操作、股票、数量、状态在卡片内保持协调间距。
- 交易记录与更新时间统一走后端时间解析辅助函数：无时区时间按 UTC 解释后再转上海时区，避免仪表盘上出现 `05:12` 这类偏移显示。
- 当前表头和数据行统一改为居中基线对齐，避免不同列因左右对齐规则不同造成视觉偏移。
- 关键字段增加悬停提示，状态与数量采用更紧凑的展示方式，提升 400px 卡片宽度下的可读性。
- 新成交插入时对首行增加短时高亮动效（约 280ms），提示有新数据但不打断阅读。
- 当首行出现新成交时显示 `NEW` 标记，保持约 2 秒后淡出，帮助快速定位新增记录。
- 离线场景改为“弱提示”：
  - 保留上次成功数据，不立即清空列表。
  - 底部显示“数据更新稍有延迟”与“最后更新 HH:mm:ss”。
  - 提供“立即刷新”按钮用于手动重试。
- 实时口径修复（2026-03-09 更新）：
  - `TradeRecordsCard` 已恢复自动刷新（接入 `refreshOrchestrator`）；
  - 数据口径改为“成交优先、委托降级”，并在降级时显示“成交接口降级为委托记录”提示；
  - 卡片请求与全局交易模式联动（`real|simulation`），避免实盘与模拟盘记录混合。

## 策略监控模块说明（2026-02 更新）
- 策略列表采用告警优先排序：`error > running > stopped`，同状态按 `updated_at` 倒序。
- 异常策略显示“异常原因”标签（优先 `error_message`，其次 `error_code/description`）。
- 卡片底部新增弱提示状态条与“最后更新时间”，服务抖动时保持已有数据，不再使用强打断提示。
- 当前模块定位为“监控展示”，仅展示统计与状态，不提供策略启动/停止操作。
- 异常策略提供“查看详情”快捷入口：点击后写入 `sessionStorage.dashboard_strategy_focus_id` 并跳转策略页。
- 支持“查看所有策略”放大展开动效（轻量模态缩放）：
  - 普通卡片可平滑放大为沉浸式扩展面板。
  - 扩展态显示更多策略条目（最多 12 条）并提供右上角关闭按钮。
  - 扩展态主按钮变为“进入策略中心”。
  - 扩展态禁用整卡 hover 变灰，改为内容行 hover 时浅蓝高亮。
  - 扩展态内部统计区/列表区/状态区补充圆角容器，视觉层次更统一。

## 策略监控模块对齐修复（2026-03-19）

- `StrategyMonitorCard` 已恢复自动刷新，监控卡不再停留在手动刷新模式。
- `StrategyMonitorCard` 会读取全局 `tradingMode` 并传入 `useStrategies`，仪表盘在实盘/模拟盘切换时策略监控同步切换数据口径。
- 后端列表返回的最近一次回测摘要会回填到策略卡，用于展示 `total_return`、`today_return`、`today_pnl`、`risk_level` 和异常信息；其中 `today_return` / `today_pnl` 在实盘运行时会优先使用 trade 服务的实时组合快照。
- 策略详情列表右侧新增“今日盈亏”徽章，方便快速识别绝对盈亏，不再只看百分比。
- 运行态展示口径收敛为 `running / starting / stopped / error`，界面中的“暂停”统计项已改为“已停止”，避免与后端生命周期状态混淆。

## 策略监控模块紧凑布局（2026-03-20）

- 压缩卡片上下间距：头部、统计区、收益区、状态栏与底部按钮的 `margin/padding` 统一下调。
- 策略列表行高与行间距收紧（名称/标签/今日盈亏徽章区域更紧凑），同高度下可显示更多条目。

## 信息通知模块说明（2026-02 更新）
- 支持“查看所有通知”弹窗展开动效（轻量模态缩放，与策略模块一致）。
- 扩展态展示更多通知条目（最多 15 条），并保留逐条已读标记交互。
- 扩展态提供“全部标记已读”和右上角关闭按钮。

- 扩展态内部统计区/分类区/列表区补充圆角容器，减少矩形边缘割裂感。
- 展开态滚动续拉（2026-03-13）：

  - 列表头显示“已加载 X / 总数 Y”，避免总数与可见条数理解偏差；
  - 保留“加载更多”按钮作为滚动触发的手动兜底。
  - 统计口径统一为“近 7 天通知总数”（`days=7`），不展示全历史累计总量。
  - 弹窗右上角新增“一键清除通知”按钮：调用后端 `POST /api/v1/notifications/clear` 清除当前窗口（默认近 7 天）通知。

## 信息通知中心收敛（2026-03-09 更新）
- `NotificationQuickCard` 的“前往”按钮已改为跳转真实通知中心 Tab（`notifications`），不再跳转策略页。
- `NotificationCenter` 已收敛为单一后端数据源（`useNotifications + userService`）：
  - 支持筛选：全部/未读/系统/交易/市场/策略；
  - 支持单条已读、全部已读、`action_url` 跳转；
  - 使用统一字段口径：`title/content/is_read/created_at/action_url`（兼容历史别名）。
- 实时通知更新（2026-03-10）：
  - 通知中心与快速卡片统一复用 `useNotifications`；
  - 默认订阅 `notification.{user_id}`，收到增量通知后头插列表并更新未读数；
  - WS 断开时显示“已切换为轮询刷新”，不中断已有数据展示。
- 展示策略补充：
  - 仪表盘 `NotificationQuickCard` 仅拉取并展示最新 10 条通知；
  - 快速卡片折叠态仍仅展示前 3 条，展开态支持滚动续拉（默认首批 20 条）；
  - 独立 `NotificationCenter` 默认展示最近 7 天通知，仍按时间倒序排列。
  - 独立通知中心内点击通知 `action_url=/backtest` 时，不走普通路由跳转，而是直接切到“回测中心 > 回测历史”。
  - 若通知未显式提供 `action_url`，独立通知中心会按 `type + 标题/内容关键词` 自动推断落点：
    - 回测/策略完成类：跳“回测中心 > 回测历史”；
    - 交易/订单/成交类：跳“实盘交易”；
    - 市场/行情类：跳“仪表盘”；
    - 账户/安全/登录类：跳“个人中心”。
  - 仪表盘 `NotificationQuickCard` 也复用同一套跳转推断规则；可跳转的通知项会显示右侧箭头并支持点击直达。
  - 可跳转通知会补充 hover 提示文案，例如“跳转到回测历史/交易页/个人中心”，降低用户试错成本。

## 智能图表模块说明（2026-02 更新）
- 布局改为“固定骨架 + 动态填充”：
  - 上方固定每日收益坐标轴区域（无数据时仍显示轴线与刻度）。
  - 下方左侧交易次数、右侧持仓占比两个固定图槽。
- 渲染策略：
  - 有数据时填充对应图表。
  - 无数据时保留容器并显示占位文案，不再整卡空白。
- 状态提示：
  - 底部展示连接状态、数据延迟提示与“最后更新时间”。
- 数据拉取策略（2026-03-09 更新）：
  - `IntelligenceChartsCard` 已恢复默认自动拉取，不再在组件层硬编码 `autoRefresh:false`。
  - 首屏进入仪表盘会直接触发读库请求，失败时按空态文案降级，不中断卡片渲染。
- 每日收益图补充：
  - 固定最近 30 日横坐标。
  - 盈利使用红色柱体显示在零轴上方，亏损使用绿色柱体显示在零轴下方。
  - 移除背景网格线，提升画面简洁度与读数聚焦。
  - 实盘模式下每日收益优先使用后端账本日快照（`real_account_ledger_daily_snapshots`）生成曲线；账本未生成时再回退到历史收益接口，避免短时快照抖动造成曲线来回跳。
  - 当前文案与数值口径已统一为“每日收益率”，tooltip 与纵轴均按 `%` 展示，不再用金额口径冒充收益率。
- 交易次数图补充：
  - 固定展示最近 7 天交易次数，不再显示更长时间窗口。
  - 纵轴改为整数步进，避免出现 1.2、1.6 这类小数刻度。
  - 图表标题改为“近7日交易次数”，与实际展示周期一致。

## 资金概览模块说明（2026-03-23 更新）
- `FundOverviewCard` 已恢复自动刷新（接入 `useFundData` 默认自动拉取 + 协调器刷新）。
- 收益率展示统一使用服务层返回的 `totalReturn`，移除组件内重复计算，避免双口径漂移。
- 账户展示文案已进一步收敛为 `初始权益 / 今日盈亏 / 本月盈亏 / 总盈亏 / 日收益率 / 总收益率`：
  - 实盘模式的“初始权益”统一按 `当前权益 - 总盈亏` 计算；
  - `dailyReturn` 优先消费 `/account` 规范化字段 `daily_return_ratio / daily_return_pct`，不再由组件按总资产临时估算。
- “本月盈亏”改为真实字段优先：仅在后端提供 `monthlyPnL` 时展示数值；无数据时显示 `--`，不再用总盈亏比例估算。
- 当用户选择实盘且实盘账户未上报（`is_online=false`）时，卡片维持实盘口径，不再回退模拟盘，并展示“实盘未上报”。
- 当后端标记盈亏指标不可用时（`*PnLAvailable=false` / `totalReturnAvailable=false`），对应数值显示 `--`，避免把未知态渲染成 0。
- `initialCapital` 现在统一按 `totalAsset - totalPnL` 计算；仅在总盈亏缺失或异常时才回退其他兜底值。`initialCapitalEstimated=true` 时，标题会展示“初始权益(估算)”。
- 底部状态提示已改为固定安全区容器，统一增加内边距与最小高度，避免“模拟数据/实时更新”文案在卡片右下角被遮挡。
