# analysis

用途：分析相关模块。

## 说明
- 归属路径：electron\src\components\backtestCenter\analysis
- 修改本目录代码后请同步更新本 README

## 近期更新
- 基础风险面板视觉细化（2026-04-27）：
  - “每日收益率”与“收益率分布”两张卡片统一为固定总高度，图表区域改为自适应伸缩，避免右侧内容变化导致卡片高度不齐；
  - 收益率分布图纵轴标题补充独立样式与间距设置，左上角文案显示更协调。
- 分析面板配色统一：
  - `BasicRiskPanel`、`PerformancePanel`、`TradeStatsPanel`、`PositionPanel` 统一收敛到 A 股风格的红涨绿跌规则；
  - 收益相关指标优先使用红色，回撤/负向风险指标使用绿色，中性配置类指标改为灰色或暖色系。
- 交易统计面板增强：
  - 将“盈亏比”主展示替换为“盈亏天数比”（盈利交易日/亏损交易日），降低在无真实单笔 `pnl` 场景下的口径歧义。
  - “持仓天数分布”固定展示 6 个常用周期：`<=1天`、`1-7天`、`7-30天`、`30-90天`、`90-180天`、`180-365天`。
  - 前端对历史自动分箱数据做兼容重映射（统一聚合到上述 6 桶），避免出现“区间1/区间2”不可读标签。
  - 仅在持仓分布无数据时，才切换为“月度交易次数分布（替代）”图表。
- 基础风险面板口径修复：
  - 移除“信息比率”卡片（后端当前链路无法稳定计算该指标）。
  - 指标卡新增非数值兜底，防止后端异常值导致页面显示 `NaN%`。
- 仅保留后端已实现分析能力对应面板：`BasicRiskPanel`、`TradeStatsPanel`、`BenchmarkPanel`。
- 持仓分析模块已从“高级分析”主入口移除，避免因后端持仓解析不稳定导致整页体验受损。
- 删除面板内“待补充”占位区块（如“相关性回归图待补充”“等待成交明细接入”），避免前端展示不可交付功能。
- 基准对比面板精简为 3 个关键指标（超额收益/Beta/Alpha），卡片宽度与上方模块保持一致。
- 数值展示统一保留 2 位小数；收益相关指标遵循 A 股颜色约定（正收益红色、负收益绿色）。
