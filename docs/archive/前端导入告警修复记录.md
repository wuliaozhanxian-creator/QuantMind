# 前端导入告警修复记录

## 背景

本次修复针对 Electron 前端在 Vite 打包时出现的静态/动态导入冲突警告，以及由此引发的 chunk 过大提示。核心目标是消除同一模块被同时静态引用和动态引用导致的 chunk 切分失效问题，并顺手修复了一个由重构引起的重复类型声明报错。

## 主要问题

1. `backtestService` 同时被静态导入和动态导入，触发 Vite 的 chunk 警告。
2. `realTradingService` 存在同样的静/动混用问题，覆盖实盘页、手动任务页、策略管理页和交易历史页。
3. Monaco 相关资源在 `lazyLoad.tsx` 与 `monaco/setup.ts` 间出现混用，导致打包器无法稳定拆分 chunk。
4. 认证页和布局页存在部分页面组件的静态聚合导出/直接导入，与现有动态路由加载链路冲突。
5. `RealTradingPage.tsx` 出现 `PreflightCheckItem` 重复声明，导致 Babel 报错。

## 修复策略

### 1. 统一运行时服务加载方式

将回测和实盘相关的运行时服务改为按需动态导入：

- `backtestService` 仅保留类型导入，运行时统一改为 `await import(...)`
- `realTradingService` 同样改为按需加载
- 对 WebSocket 基址、轮询、任务提交、日志拉取等路径做了局部适配，避免同步访问服务对象

涉及文件包括：

- `electron/src/components/backtestCenter/EnhancedQuickBacktest.tsx`
- `electron/src/components/backtest/QlibParameterOptimizer.tsx`
- `electron/src/components/optimization/StrategyCreationModal.tsx`
- `electron/src/components/backtestCenter/EnhancedAdvancedAnalysisModule.tsx`
- `electron/src/components/backtestCenter/BacktestHistoryModule.tsx`
- `electron/src/components/optimization/GridSearchPanel.tsx`
- `electron/src/pages/trading/tabs/ManualTaskPage.tsx`
- `electron/src/pages/trading/tabs/StrategyManagement.tsx`
- `electron/src/pages/trading/tabs/TradingHistory.tsx`
- `electron/src/pages/trading/RealTradingPage.tsx`
- `electron/src/hooks/useBacktestQueries.ts`
- `electron/src/stores/backtestStore.ts`

### 2. 统一 Monaco / 认证页加载链路

为避免 `@monaco-editor/react` 和 `@monaco-editor/loader` 的静/动混用，移除了 `lazyLoad.tsx` 中会主动预热 Monaco 组件的动态 import，只保留静态资源预热逻辑。

同时将认证路由统一切回 `lazyLoad.tsx` 提供的懒加载组件，避免同一页面同时从聚合导出和动态路由入口被加载。

涉及文件包括：

- `electron/src/features/auth/utils/lazyLoad.tsx`
- `electron/src/features/auth/AppRoutes.tsx`
- `electron/src/App.tsx`
- `electron/src/components/layout/DashboardLayout.tsx`

### 3. 调整打包分包策略

在 `electron/vite.config.ts` 中加入 `manualChunks`，将常用大依赖拆分为独立 vendor chunk，降低主包体积并缓解 chunk size 告警。

## 结果

### 已消除的问题

- `backtestService` 的静/动导入冲突告警已清除。
- `realTradingService` 的静/动导入冲突告警已清除。
- Monaco 相关静/动混用告警已清除。
- 认证页和布局页引发的页面级静/动混用告警已清除。
- `RealTradingPage.tsx` 的重复 `PreflightCheckItem` 声明错误已修复。

### 当前构建状态

前端构建可以正常通过，当前仅保留 Vite 的通用 chunk size 提示；该提示不影响构建成功，但说明仍存在较大的 vendor chunk，可在后续继续按需优化。

```bash
cd /Users/qusong/git/quant/electron && npm run build:react
```

构建结果：成功。

## 备注

本次修复以“统一动态导入 + type-only import”为主线，目标是让同一模块只走一条加载链路，避免 Vite/Rollup 无法正确重排 chunk。后续如果还要继续压缩主包体积，可以进一步细化 `manualChunks` 或拆分更重的第三方依赖。
