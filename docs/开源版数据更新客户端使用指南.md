# QuantMind 开源版数据更新客户端使用指南

本文档面向开源版用户，说明如何通过 QuantMind 官方数据更新接口增量更新本地：

- `db/qlib_data`
- `db/feature_snapshots`
- `stock_daily_latest` 相关离线文件

客户端脚本位于：

```bash
scripts/data/update_client.py
```

## 一、更新机制

开源客户端不会直接访问腾讯云 COS 私有 Bucket，也不会持有 COS 密钥。更新流程如下：

```text
客户端
  -> 携带 X-Access-Key / X-Secret-Key 请求 quantmind-api
  -> quantmind-api 校验 API Key 与订阅 data_access 权限
  -> quantmind-api 返回短时效 COS signed URL
  -> 客户端下载更新包
  -> 客户端校验 sha256
  -> 客户端解包到本地 db/ 目录
  -> 客户端记录已应用版本
```

COS 上每天只存少量归档包，不逐个分发 `qlib_data` 的 30000+ 小文件。典型 manifest 如下：

```json
{
  "version": "20260504",
  "trade_date": "2026-05-04",
  "files": [
    {
      "name": "qlib_data_delta_20260504.tar.zst",
      "kind": "qlib_data",
      "target": "db/qlib_data",
      "size": 31457280,
      "sha256": "xxx",
      "cos_key": "data-updates/20260504/qlib_data_delta_20260504.tar.zst"
    }
  ]
}
```

服务端返回给客户端时，`cos_key` 会被替换为短时效 `url`。

## 二、环境要求

### Python

建议使用 Python 3.10+。

脚本只使用 Python 标准库，不需要安装腾讯云 SDK。

### 系统命令

处理 `tar.zst` 归档需要：

```bash
tar
zstd
```

macOS 安装示例：

```bash
brew install zstd
```

Ubuntu / Debian 安装示例：

```bash
sudo apt-get update
sudo apt-get install -y zstd tar
```

Windows 用户建议在 WSL 中运行。

## 三、配置 API Key

在 QuantMind 个人中心或订阅后台获取：

- `access_key`
- `secret_key`

该 API Key 必须具备以下权限之一：

- `data_access`
- `data_sync`

同时账号订阅套餐必须包含：

```json
{
  "data_access": true
}
```

推荐通过环境变量配置，避免把密钥写入命令历史：

```bash
export QUANTMIND_ACCESS_KEY="qm_live_xxx"
export QUANTMIND_SECRET_KEY="sk_xxx"
```

如使用私有部署或测试环境，可覆盖 API 地址：

```bash
export QUANTMIND_UPDATE_API_BASE="https://api.quantmind.cloud"
```

默认 API 地址为：

```text
https://api.quantmind.cloud
```

## 四、首次 dry-run

在项目根目录运行：

```bash
python scripts/data/update_client.py --dry-run
```

dry-run 会完成：

- 请求最新版本 manifest
- 校验 API Key 与订阅权限
- 展示将要下载的文件
- 不下载、不解压、不修改本地数据

示例输出：

```text
[quantmind-update] version: 20260504
[quantmind-update] trade_date: 2026-05-04
[quantmind-update] files: 2
[quantmind-update] plan: qlib_data qlib_data_delta_20260504.tar.zst -> db/qlib_data (31457280 bytes)
[quantmind-update] plan: feature_snapshots feature_snapshots_delta_20260504.tar.zst -> db/feature_snapshots (10485760 bytes)
[quantmind-update] dry-run complete; no files downloaded or applied
```

## 五、应用最新更新

```bash
python scripts/data/update_client.py
```

脚本会自动：

1. 请求 `/api/v1/data-updates/latest`
2. 下载 signed URL 指向的更新包
3. 校验每个文件的 `sha256`
4. 解压 `.tar.zst` 到项目根目录
5. 对普通 `.zst` 文件按 `target` 解压或复制
6. 更新 `.quantmind_data_update_state.json`

默认状态文件：

```text
.quantmind_data_update_state.json
```

内容示例：

```json
{
  "applied_updates": [
    "20260503",
    "20260504"
  ],
  "current_version": "20260504",
  "last_applied_at": "2026-05-04T18:30:01+0800"
}
```

脚本再次运行时，如果最新版本已应用，会自动跳过。

## 六、应用指定版本

```bash
python scripts/data/update_client.py --version 20260504
```

该命令请求：

```text
GET /api/v1/data-updates/20260504
```

适合补装某一天的增量包。

## 七、常用参数

```bash
python scripts/data/update_client.py \
  --api-base https://api.quantmind.cloud \
  --project-root /path/to/quantmind \
  --version latest \
  --timeout 120
```

参数说明：

| 参数 | 说明 |
| --- | --- |
| `--api-base` | `quantmind-api` 地址，默认读取 `QUANTMIND_UPDATE_API_BASE` 或 `https://api.quantmind.cloud` |
| `--access-key` | API Access Key，默认读取 `QUANTMIND_ACCESS_KEY` |
| `--secret-key` | API Secret Key，默认读取 `QUANTMIND_SECRET_KEY` |
| `--project-root` | 本地 QuantMind 项目根目录，默认当前目录 |
| `--version` | `latest` 或指定版本号 |
| `--state-file` | 自定义状态文件路径 |
| `--timeout` | HTTP 超时时间，默认 120 秒 |
| `--dry-run` | 只查看计划，不下载、不解包 |
| `--force` | 即使版本已记录为应用，也强制重新应用 |
| `--keep-downloads` | 保留下载包到 `.quantmind_update_tmp/`，便于排查 |

## 八、归档包路径约定

推荐服务端生成的 `.tar.zst` 包内路径使用项目根目录相对路径，例如：

```text
db/qlib_data/calendars/day.txt
db/qlib_data/features/SH600000/open.day.bin
db/qlib_data/instruments/all.txt
db/feature_snapshots/model_features_2026.parquet
```

客户端会直接执行：

```bash
tar --zstd -xf <archive> -C <project-root>
```

因此归档包内不要包含绝对路径，也不要包含 `../`。

客户端在解包前会检查 tar 成员路径，发现以下路径会拒绝执行：

- `/absolute/path`
- `../outside`
- `db/qlib_data/../../outside`

## 九、自动化定时更新

Linux crontab 示例，每天 18:30 更新：

```cron
30 18 * * 1-5 cd /opt/quantmind && QUANTMIND_ACCESS_KEY=qm_live_xxx QUANTMIND_SECRET_KEY=sk_xxx /usr/bin/python3 scripts/data/update_client.py >> logs/data_update_client.log 2>&1
```

macOS 可使用 `launchd` 或 crontab。建议将日志写入：

```text
logs/data_update_client.log
```

## 十、错误排查

### 401 Missing X-Access-Key or X-Secret-Key

未提供 API Key：

```bash
export QUANTMIND_ACCESS_KEY="qm_live_xxx"
export QUANTMIND_SECRET_KEY="sk_xxx"
```

### 401 Invalid API key

常见原因：

- `access_key` 写错
- `secret_key` 写错
- API Key 已禁用
- API Key 已过期

### 403 API key lacks data update permission

API Key 权限不包含：

- `data_access`
- `data_sync`

请在后台生成专属数据同步 Key。

### 403 Subscription does not include data_access

账号当前有效订阅不包含数据更新权限，需要升级或续费包含 `data_access=true` 的套餐。

### required command not found: zstd

安装 zstd：

```bash
brew install zstd
```

或：

```bash
sudo apt-get install -y zstd
```

### sha256 mismatch

下载文件与 manifest 不一致。常见原因：

- 网络中断导致下载不完整
- COS 对象被覆盖但 manifest 未更新
- 本地磁盘写入异常

处理方式：

```bash
python scripts/data/update_client.py --force
```

如果仍失败，应联系 QuantMind 数据服务方重新发布该版本。

## 十一、开源分发建议

开源版仓库建议包含：

```text
scripts/data/update_client.py
DOCS/开源版数据更新客户端使用指南.md
```

不要在开源仓库中包含：

- 腾讯云 COS SecretId
- 腾讯云 COS SecretKey
- 私有 Bucket 名称
- `.env`
- 用户 API Key
- 已下载的数据包
- `.quantmind_update_state.json`
- `.quantmind_update_tmp/`

`.gitignore` 建议包含：

```gitignore
.quantmind_data_update_state.json
.quantmind_update_tmp/
logs/data_update_client.log
```
