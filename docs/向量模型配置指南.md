# 向量模型配置指南

本指南详细说明了 QuantMind AI Strategy 链路中向量模型（Vector Model）的配置、启动逻辑及日常维护说明。

---

## 1. 核心功能

向量模型在系统中承担以下关键任务：
1. **语义路由 (Semantic Routing)**：将用户的自然语言需求匹配到最接近的策略原型（如：价值投资、成长性、技术分析等）。
2. **Schema RAG (检索增强生成)**：根据用户查询，从数据库中检索最相关的表结构和字段，辅助 LLM 生成精准的 DSL 或 SQL。

## 2. 快速配置（3 步）

### 第 1 步：获取 API Key
1. 登录 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)。
2. 进入「API-KEY 管理」→「创建新的 API-KEY」。
3. 复制生成的 Key（格式如 `sk-xxxxxxxxxxxxxxxx`）。

### 第 2 步：配置环境变量
在项目根目录的 `.env` 文件中添加或修改：

```bash
# DashScope 向量模型（语义理解）
DASHSCOPE_API_KEY=sk-your-real-api-key-here

# 启动预热开关（生产环境建议设为 true）
AI_STRATEGY_WARMUP=false
```

### 第 3 步：重启服务
```bash
docker-compose restart engine
```

## 3. 关键环境变量参考

| 变量名 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `AI_STRATEGY_WARMUP` | `false` | **核心开关**。是否在启动时强制预热。设为 `true` 可提升首个请求响应速度。 |
| `DASHSCOPE_API_KEY` | - | 阿里云 DashScope API 密钥。 |
| `DASHSCOPE_EMBEDDING_MODEL` | `text-embedding-v4` | 指定使用的向量模型版本。 |
| `ENV` | `dev` | 环境标识。在 `prod` 且开启预热时，若初始化失败会直接阻断进程启动。 |

## 4. 启动模式说明

### 4.1 强制预热模式 (`AI_STRATEGY_WARMUP=true`)
*   **行为**：在服务启动阶段，同步阻塞执行模型初始化。
*   **优点**：保证首个用户请求响应迅速，避免因为临时拉取 Embedding 导致超时。
*   **失败处理**：若初始化失败，服务状态将标记为 `degraded`。在生产环境 (`ENV=prod`) 下会直接抛出异常阻断启动。

### 4.2 跳过预热模式 (`AI_STRATEGY_WARMUP=false`)
*   **行为**：服务快速启动，不进行模型初始化。
*   **缺点**：用户首次发起 AI 选股请求时，会触发耗时较长的懒加载过程。

## 5. 健康检查与验证

可以通过以下接口实时监控向量模型状态：

*   **API 接口**：`GET /api/v1/health`
*   **返回示例**：
    ```json
    {
      "status": "healthy",
      "startup_health": {
        "status": "healthy",
        "warmup_completed": true,
        "warmup_duration": 15.42,
        "errors": [],
        "uptime": 120.5
      }
    }
    ```

## 6. 字段映射标准

系统统一了以下核心字段的命名口径，在维护数据库或生成 SQL 时应严格遵循：

*   **股票识别**：`symbol` (代码), `name` (简称)
*   **核心指标**：`close` (收盘价), `amount` (成交额), `total_mv` (总市值)
*   **指数成分**：`idx_hs300`, `idx_zz500`, `idx_zz1000`
*   **市值单位**：**统一为“亿元”**。
