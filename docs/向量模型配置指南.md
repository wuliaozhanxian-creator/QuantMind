# 向量模型配置指南

QuantMind 使用阿里云 DashScope Embedding（`text-embedding-v4`）进行语义理解，提升策略生成和股票筛选的准确率。

## 当前默认行为

> **未配置时，系统降级为关键字/规则匹配，不影响基础功能，但语义理解能力受限。**

| 向量功能 | 无 API Key 时 | 有 API Key 时 |
|----------|--------------|--------------|
| Schema 字段语义匹配 | 关键词匹配 | 向量余弦相似度匹配 |
| 策略意图向量解析 | 规则兜底 | 语义原型向量匹配 |
| 自然语言→字段映射 | 硬编码名映射 | embedding 语义检索 |

## 前提条件

- 已部署 QuantMind（Docker 或服务器）
- 有阿里云 DashScope 账号和 API Key

## 快速配置（3 步）

### 第 1 步：获取 API Key

1. 登录 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
2. 进入「API-KEY 管理」→「创建新的 API-KEY」
3. 复制生成的 Key（格式如 `sk-xxxxxxxxxxxxxxxx`）

### 第 2 步：配置环境变量

**方式一：修改 `.env` 文件（推荐）**

在项目根目录的 `.env` 文件中添加：

```bash
# DashScope 向量模型（语义理解）
DASHSCOPE_API_KEY=sk-your-real-api-key-here
```

**方式二：修改 `docker-compose.yml`**

在 `quantmind` 服务的 `environment` 中找到：

```yaml
- DASHSCOPE_API_KEY=${DASHSCOPE_API_KEY:-mock-api-key-not-configured}
```

将 `mock-api-key-not-configured` 替换为真实 Key。

### 第 3 步：重启服务

```bash
docker restart quantmind
```

重启后，向量模型自动预热并生效。可在日志中确认：

```bash
docker logs quantmind | grep -i "warmup\|embedding\|vector"
```

成功标志：
```
Warmup completed for vector parser and schema retriever in X.XXs
```

失败标志（无影响，降级运行）：
```
Warmup skipped/failed: DASHSCOPE_API_KEY 未配置真实密钥
```

## 环境变量参考

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DASHSCOPE_API_KEY` | `mock-api-key-not-configured` | **【必填】** DashScope API Key，同时用于 LLM 对话和向量 embedding |
| `DASHSCOPE_EMBEDDING_MODEL` | `text-embedding-v4` | embedding 模型名称 |
| `DASHSCOPE_EMBEDDING_TIMEOUT` | `60` | embedding 请求超时（秒） |
| `DASHSCOPE_BASE_URL` | `https://dashscope.aliyuncs.com/compatible-mode/v1` | API 地址（一般无需修改） |

> **注意**：`DASHSCOPE_API_KEY` 同时用于 **LLM 对话**（策略生成、代码生成）和 **向量 embedding**（语义理解）。一个 Key 覆盖全部功能。

## 关键说明

### 与个人中心 API Key 的区别

| 配置途径 | 对 LLM 对话有效 | 对向量 embedding 有效 |
|----------|:---:|:---:|
| 个人中心 → API Key 配置 | ✅ | ❌ |
| 环境变量 `DASHSCOPE_API_KEY` | ✅ | ✅ |

向量模型 **必须通过环境变量配置**，个人中心的 Key 不会生效。

### 费用

DashScope `text-embedding-v4` 按 token 计费。每次启动预热约消耗 30-50 次 embedding 调用（schema 字段 + 策略原型向量），后续每次策略生成/股票筛选按需调用。日常使用成本极低。

## 验证方法

### 1. 验证 embedding 健康检查

```bash
docker exec quantmind python -c "
from backend.services.engine.ai_strategy.services.selection.dashscope_client import DashScopeClient
c = DashScopeClient()
print('Embedding model:', c.health())
"
```

输出：`Embedding model: text-embedding-v4`

### 2. 验证语义匹配效果

在策略向导中输入自然语言描述（如 "筛选沪深300中市盈率低于15的半导体股票"），观察：
- DSL 字段映射是否准确（如 "市盈率" → `pe_ttm`）
- 行业匹配是否识别到 `industry` 列

### 3. 查看错误（如有）

```bash
docker logs quantmind | grep -i "embedding"
```

常见错误及解决：

| 错误 | 原因 | 解决 |
|------|------|------|
| `DASHSCOPE_API_KEY 未配置真实密钥` | Key 未配置或仍为 mock | 检查 `.env` 文件 |
| `401 Unauthorized` | Key 无效或过期 | 重新生成 Key |
| `Connection timeout` | 网络不通 | 检查服务器是否能访问 `dashscope.aliyuncs.com` |
| `Warmup skipped/failed` | embedding 初始化失败 | 检查日志中的具体错误信息 |

## FAQ

**Q: 不配置向量模型会影响使用吗？**
A: 不影响核心功能。策略生成和股票筛选仍可正常使用，只是语义理解依赖关键字/规则匹配而非向量相似度。

**Q: 可以用其他向量模型吗？**
A: 当前仅支持 DashScope `text-embedding-v4`。如需其他模型（如 OpenAI embedding），需修改 `dashscope_client.py`。

**Q: 向量模型和 LLM 模型有什么区别？**
A: LLM 模型（如 `qwen3.6-plus`）用于生成策略代码和对话，向量模型用于文本语义相似度计算（将文本转为向量后进行匹配）。两者使用同一个 `DASHSCOPE_API_KEY`。
