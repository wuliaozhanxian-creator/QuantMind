> **文档版本**：v2026.05.05  
> **更新摘要**：标准化市值/资金流单位，校准多周期收益率计算逻辑，补全 2026 全量历史因子。

## 一、 数据表定位
`stock_daily_latest` 是 QuantMind 投研平台与回测引擎的核心“事实表”。它集成了行情、财务、技术指标与机器学习因子，为前端 UI 展示和后端 AI 策略生成提供统一口径。

## 二、 核心字段定义与单位标准

### 1. 基础行情 (Price & Volume)
| 字段名 | 单位 | 计算/口径说明 |
| :--- | :--- | :--- |
| `open/high/low/close` | 元 | **后复权价格**。用于回测与收益率计算。 |
| `adj_factor` | - | 复权因子。实际价格 = 后复权价格 / 复权因子。 |
| `volume` | 手 | 原始成交量。休市日/停牌日该值为 0 或 NULL。 |
| `amount` | 元 | 成交额。**注意：** 前端展示时通常转换为“亿元”。 |
| `pct_change` | % | 当日涨跌幅。计算方式：`(Close_T / Close_T-1) - 1`。 |

### 2. 市值与财务 (Valuation & Finance)
| 字段名 | 单位 | 计算/口径说明 |
| :--- | :--- | :--- |
| `total_mv` | 元 | **总市值**。数据库存储为“元”。AI 策略解析时已统一按 `10^8`（亿元）作为基准过滤。 |
| `float_mv` | 元 | 流通市值。 |
| `pe_ttm / pb` | 倍 | 市盈率(TTM) / 市净率。 |
| `roe` | % | 净资产收益率。存储为百分比数值（如 15.5 代表 15.5%）。 |

### 3. 资金流向 (Money Flow)
| 字段名 | 单位 | 计算/口径说明 |
| :--- | :--- | :--- |
| `main_flow` | 元 | 主力资金净流入。前端展示建议转换为“万元”。 |
| `flow_net_amount` | 元 | 资金净流入总额。 |

### 4. 统计与技术指标 (Technicals)
| 字段名 | 周期 | 计算逻辑 |
| :--- | :--- | :--- |
| `return_Nd` | N日 | **滚动收益率**。计算：`(Close_T / Close_T-N) - 1`。已补全 5/10/20/60 日序列。 |
| `volume_ma_N` | N日 | **成交量均线**。计算：`AVG(volume) OVER(ORDER BY trade_date ROWS N-1 PRECEDING)`。 |
| `volume_ratio_N` | - | **量比**。计算：`volume / volume_ma_N`。反映成交量异动。 |

## 三、 投研平台关键逻辑说明

### 1. “次日/3日收益”计算规约
为确保与 K 线走势完全一致，系统不再使用表内预计算字段，而是采用实时计算：
*   **基准日 (T)**：模型生成信号的交易日。
*   **涨跌幅 (当日)**：`(Close_T / Close_T-1) - 1`。
*   **次日收益**：`(Close_T+1 / Close_T) - 1`。
*   **3日收益**：`(Close_T+3 / Close_T) - 1`。

### 2. 节假日处理 (Trading Calendar)
系统通过 `WHERE volume > 0` 自动过滤非交易日。
*   **影响**：`LEAD(1)` 将跳过节假日，准确指向下一个真实的交易日。
*   **维护建议**：导入数据时，务必确保非交易日的 `volume` 设为 0 或不入库。

## 四、 数据维护与质量监控

### 1. 每日增量更新流程
1. 每日 16:00 触发行情下载。
2. 写入 `stock_daily_latest`。
3. **关键：** 运行 `scripts/fill_sdl_from_parquet.py` 补全滚动收益率与量均线。
4. 检查 `stock_name` 是否因更名需更新（`UPDATE ... FROM ...`）。

### 2. 异常审计标准
若出现以下情况，需介入排查：
*   `close` 字段出现 0 或负数（数据源异常）。
*   `total_mv` 数量级偏移（需核对是否误入“万元”单位）。
*   `return_5d` 大量缺失（说明补全脚本未执行）。
