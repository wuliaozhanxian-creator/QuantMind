# QuantMind 数据配置指南

本指南介绍 QuantMind 项目需要维护的 4 组核心数据，包括数据来源、生成方式、目录结构和日常更新流程。

---

## 数据总览

| # | 数据组 | 目录/存储 | 用途 | 数据来源 |
|---|--------|-----------|------|----------|
| 1 | 实时行情数据 | Redis (DB 3) | 模拟盘权益计算、实盘模拟盘调仓预案 | 通达信推送 |
| 2 | Qlib 回测数据 | db/qlib_data/ | Qlib 引擎回测 | qlib 官方工具转换 |
| 3 | 基础对齐数据 | db/custom/fundamental_aligned.parquet | 回测参数扩展、投研平台数据源 | 通达信 |
| 4 | 模型特征快照 | db/feature_snapshots/ | 模型训练与推理 | Level2 数据 + 基础数据 + CSMAR |

4 组数据的依赖关系：

```
通达信 ──(实时推送)──→ Redis 行情快照
     │
     └──(导出)──→ fundamental_aligned.parquet ──→ stock_daily_latest 表

CSMAR 数据 ──(银层构建)──→ market_silver_v2/ ──┐
                                               ├──→ feature_snapshots/
Level2 逐笔数据 ──────────────────────────────┘
     │
     └──(qlib 转换)──→ qlib_data/
```

## 0. 原始数据源

所有脚本依赖的原始数据均为用户手动下载，无自动下载脚本。以下是需要用户准备的原始数据：

| 原始数据 | 存放路径 | 获取方式 | 用于 |
|----------|----------|----------|------|
| CSMAR 数据 | db/csmar/ | CSMAR 官网手动下载 | 银层构建、qlib 重建、L1 特征计算 |
| Level2 逐笔数据 | baidudesk/ | 手动获取 | L2 高频特征计算 |
| 通达信行情 | 通达信客户端 TQ 接口 | 通达信专业普及版 TQ 策略模块 | 实时行情推送、fundamental_aligned.parquet |
| 公司基本信息 | 脚本同目录 | CSMAR 官网下载「公司文件」完整数据 | full_update.py 公司信息填充 |
| 指数成分股 | 脚本同目录 | CSMAR 官网下载成分股列表 | full_update.py 指数标记 |

### 0.1 CSMAR 数据

从 CSMAR 中转站（https://www.chaoslib.com/）手动下载，解压后放入 `db/csmar/` 目录。152 维完整特征快照共需 13 张数据表，详见第 4 节 4.6.1。基本数据表包括：日个股回报率、复权因子表、换手率表、指数日行情、公司文件。

**费用**: CSMAR 中转站订阅约 20 元/月。

CSMAR 数据按日期子目录组织，例如 `db/csmar/20260524/`。每次下载后解压到新日期子目录。

### 0.2 Level2 逐笔数据

Level2 逐笔数据需要用户手动获取，放入 `baidudesk/` 目录。每天需要两个文件：

- `baidudesk/snapshot_YYYYMMDD.parquet` — Level2 快照数据（十档行情）
- `baidudesk/deal_YYYYMMDD.parquet` — Level2 逐笔成交数据

**获取渠道**: 淘宝搜索「龙头学堂」店铺购买，通过百度网盘分享。

**费用**: Level2 数据约 50 元/月（含百度网盘分享），百度网盘订阅费约 30 元/月。

### 0.3 通达信客户端

需要在 Windows 机器上安装通达信专业普及版，开通 TQ 策略模块。用于：
- 实时行情推送（`scripts/市场数据到redis.py`）
- 全量历史数据拉取（`scripts/full_update.py`）

**费用**: 通达信专业普及版订阅约 30 元/月。

## 1. 实时行情数据

### 1.1 说明

实时行情数据存储在 Redis DB 3 中，由通达信客户端通过推送脚本实时写入服务器。模拟盘权益计算和实盘调仓预案依赖此数据获取最新价格。

### 1.2 存储结构

- Redis DB: 3
- Key 格式: `market:snapshot:{symbol}` (小写前缀，如 `market:snapshot:sh600000`)
- 数据格式: JSON，包含最新价、开高低收、成交量等

### 1.3 配置要求

- Redis 地址：通过 `.env` 中 `REDIS_HOST` / `REDIS_PORT` / `REDIS_PASSWORD` 配置
- Docker 环境下默认使用容器 `quantmind-redis` (127.0.0.1:6379)

### 1.4 数据获取方式

#### 1.4.1 前置条件

- 已安装通达信客户端，并开通 **专业普及版**（TQ 策略模块需要此版本支持）
- 服务器 Redis 已正常运行

#### 1.4.2 配置环境变量

在通达信客户端所在机器上配置 `.env` 文件，或设置系统环境变量：

```bash
REDIS_HOST=<服务器IP>
REDIS_PORT=<服务器端口>
REDIS_PASSWORD=<Redis密码>
```

#### 1.4.3 部署推送脚本

1. 登录通达信客户端
2. 在顶部菜单找到 **TQ 策略** 模块，点击进入
   - 若没有该模块，需开通通达信专业普及版
3. 将推送脚本 `scripts/市场数据到redis.py` 复制到通达信安装目录下的 PYPlugins/user 目录：

   ```
   C:\new_tdx64\PYPlugins\user\市场数据到redis.py
   ```

4. 在 TQ 策略管理器中运行该脚本

脚本启动后会自动循环获取全市场股票快照并推送到 Redis，每 10 秒推送一次，仅在交易时间段内运行（09:15-11:30, 13:00-15:00）。

### 1.5 验证

```bash
# 检查 Redis 中是否有行情数据
docker exec -it quantmind-redis redis-cli -n 3 KEYS "market:snapshot:*" | head -10

# 查看单只股票快照
docker exec -it quantmind-redis redis-cli -n 3 GET "market:snapshot:sh600000"
```

## 2. Qlib 回测数据

### 2.1 说明

`db/qlib_data/` 是 Qlib 引擎的回测数据，采用 Qlib 原生二进制格式。需要准备 **2 组口径**的数据：

| 口径 | 用途 | 说明 |
|------|------|------|
| 后复权 | 收益率计算、技术指标计算 | 价格序列连续可比，适合策略回测 |
| 不复权 | 真实成交价格 | 反映实际交易成本，适合模拟下单 |

### 2.2 目录结构

```
db/qlib_data/
├── calendars/
│   └── day.txt              # 交易日历 (2016-01-04 ~ 至今)
├── instruments/
│   ├── all.txt              # 全部标的 (~5548)
│   ├── csi300.txt           # 沪深300成分股 (300)
│   ├── csi500.txt           # 中证500成分股 (500)
│   ├── csi1000.txt          # 中证1000成分股 (1000)
│   ├── csi800.txt           # 中证800成分股 (799)
│   └── margin.txt           # 融资融券标的 (~3496)
└── features/
    ├── sh600000/            # 个股目录 (5545+)
    │   ├── open.day.bin
    │   ├── high.day.bin
    │   ├── low.day.bin
    │   ├── close.day.bin
    │   ├── volume.day.bin
    │   ├── amount.day.bin
    │   ├── factor.day.bin   # 复权因子
    │   ├── adjclose.day.bin # 后复权收盘价
    │   ├── change.day.bin
    │   └── vwap.day.bin
    ├── idx_sh000300/        # 沪深300指数
    ├── idx_sh000852/        # 中证1000指数
    └── idx_sh000905/        # 中证500指数
```

### 2.3 数据口径说明

Qlib 二进制中通过不同字段同时存储两种口径，无需生成两套目录：

| 字段 | 口径 | 计算方式 | 用途 |
|------|------|----------|------|
| open / high / low / close | 不复权 | 原始价格 | 真实成交价格参考 |
| adjclose | 后复权 | close × factor | 收益率计算、技术指标 |
| factor | 后复权因子 | 累积复权因子 | 复权基准 |
| volume | 不复权 | 原始成交量 / 100 | 成交量 |
| amount | 不复权 | 原始成交额 / 1000 | 成交额 |

后复权收益率 = adjclose.pct_change()，等价于 (close_today × factor_today) / (close_yesterday × factor_yesterday) - 1。

### 2.4 指数数据

3 个指数的二进制存放在 `features/` 目录下，结构与个股一致：

| 目录名 | 指数 | 代码 |
|--------|------|------|
| idx_sh000300 | 沪深300 | 000300.SH |
| idx_sh000905 | 中证500 | 000905.SH |
| idx_sh000852 | 中证1000 | 000852.SH |

每个指数目录包含相同的 .day.bin 文件（open/high/low/close/adjclose/factor/volume/amount/vwap/change）。指数的 factor 通常为 1.0（指数本身不复权），adjclose 等于 close。

### 2.5 生成方式

本项目（quant）暂无转换脚本，以下脚本来自企业版（quantmind），可按需复制使用。

#### 2.5.1 全量构建（首次部署）

使用 Qlib 官方 dump_bin 工具，从 parquet 生成完整 qlib_data：

**脚本**: `scripts/data/processing/convert_to_qlib_bin_official.py`

```bash
# 前置：安装 qlib
pip install pyqlib

# 准备源数据 parquet（需含 symbol, date, open, high, low, close, volume, factor 字段）
# 然后执行转换
python scripts/data/processing/convert_to_qlib_bin_official.py
```

脚本流程：
1. 读取 parquet → 按 symbol 分组导出 CSV
2. 调用 qlib 官方 `dump_bin.py dump_all` 转为二进制
3. 生成 calendars/ + instruments/ + features/ 完整目录

#### 2.5.2 混合重建（CSMAR 源数据）

从 CSMAR 原始数据 + fundamental_aligned.parquet 重建完整 qlib_data，适合需要精确控制复权因子的场景：

**脚本**: `tools/reconstruct_qlib_hybrid.py`

```bash
# 前置：需要 CSMAR 数据 + DuckDB
# 数据路径：
#   /Volumes/M2SSD/CSMAR/日个股回报率文件/  (原始价格)
#   /Volumes/M2SSD/CSMAR/股票价格复权因子表(日)/  (复权因子)
#   db/custom/fundamental_aligned.parquet  (对齐数据)

python tools/reconstruct_qlib_hybrid.py
```

脚本流程：
1. 加载 CSMAR 日个股回报率（Opnprc/Hiprc/Loprc/Clsprc/Dnshrtrd/Dnvaltrd）
2. ASOF 合并 CSMAR 复权因子表（CumulateBwardFactor）
3. 计算后复权价格：adjclose = close × factor
4. 保留现有指数二进制（idx_sh000300/idx_sh000905/idx_sh000852）
5. 生成全部 .day.bin（open/high/low/close/volume/amount/factor/adjclose/vwap/change）
6. 原子替换 features/ 目录
7. 重建 instruments/all.txt 和成分股列表

> **注意**: `fundamental_aligned.parquet` 是前复权数据，不含后复权因子（`cum_bward_factor`），不能直接用于 qlib 增量更新。qlib 数据的增量更新需要后复权因子，应从 CSMAR 复权因子表获取。

### 2.6 验证

```bash
# 检查交易日历
wc -l db/qlib_data/calendars/day.txt
head -1 db/qlib_data/calendars/day.txt
tail -1 db/qlib_data/calendars/day.txt

# 检查标的数量
wc -l db/qlib_data/instruments/all.txt

# 检查个股 features
ls db/qlib_data/features/sh600000/

# 检查指数 features
ls db/qlib_data/features/idx_sh000300/
```

## 3. 基础对齐数据

### 3.1 说明

`db/custom/fundamental_aligned.parquet` 是项目的核心数据集，深度整合了全市场标的的行情数据 (OHLCV)、深度因子 (估值/质量/技术指标)、资金流向、微观结构和行业标签。

它是整个平台的 "数据心脏"：
- **回测参数扩展**：Qlib 回测时直接调用，无需自行计算特征
- **投研平台数据源**：投研平台通过 PostgreSQL 的 `stock_daily_latest` 表读取此数据
- **统一对齐**：所有字段已做时间轴对齐，开箱即用

### 3.2 数据来源

通过通达信获取基础行情数据，生成 parquet 文件后需同步更新到 PostgreSQL 数据库的 `stock_daily_latest` 表。

### 3.3 数据规模

- 文件大小：约 518MB
- 覆盖年份：2016 ~ 至今
- 标的范围：全市场 (~5500+)
- 字段维度：OHLCV + 89 维因子 + 资金流 + 微观结构 + 行业标签

### 3.4 生成与更新流程

```
通达信客户端 (Windows)
    │
    ├─ scripts/full_update.py
    │   ├─ 拉取全市场 K 线 (前复权 + 不复权)
    │   ├─ 拉取快照 (估值/市值/换手率)
    │   ├─ 计算技术指标 (MA/RSI/KDJ/MACD/beta/收益率/连板)
    │   ├─ 清洗 (去北交所/B股/退市股)
    │   ├─ 填充公司信息 + 指数成分标记
    │   └─ 输出 fundamental_aligned.parquet (前复权)
    │
    ▼
拷贝到服务器 db/custom/fundamental_aligned.parquet
    │
    └─ scripts/data/maintenance/sync_stock_daily_latest_from_parquet.py
        └─ 增量 upsert → PostgreSQL stock_daily_latest 表
```

#### 3.4.1 生成 parquet 文件 (通达信客户端机器)

**脚本**: `scripts/full_update.py`

**运行环境**: Windows，需安装通达信专业普及版（TQ 策略模块），通达信客户端已登录

**前置文件**（放在脚本同目录）:
- CSMAR 公司文件完整数据（文件名每次下载不同，不固定）— 含 Stknme/Indnme/Nindcd/PROVINCE 字段
- 指数成分股列表（从 CSMAR 下载）:
  - 沪深300成分股（含「成份券代码」和「交易所」字段）
  - 中证500成分股
  - 中证1000成分股

```bash
# 全量拉取（默认从 20241201 开始）
python full_update.py

# 自定义起始日期
python full_update.py --start 20160101

# 跳过拉取，仅重算指标（已有临时文件 temp_full_data.parquet 时）
python full_update.py --skip-fetch
```

脚本流程:
1. 连接通达信，获取全市场股票列表（上证主板/深证主板/创业板/科创板/北交所）
2. 批量拉取 K 线数据（前复权 front + 不复权 none 双口径），每批 30 只
3. 拉取快照数据补充估值字段（pe_ttm/pb/total_mv/float_mv/turnover_rate/roe）
4. 计算 20+ 技术指标（MA/RSI/KDJ/MACD/beta_20/收益率/波动率/ATR/连板天数）
5. 清洗数据（去除北交所、B 股、close < 1 的退市股）
6. 填充公司信息（行业/省份）和指数成分股标记（idx_hs300/idx_zz500/idx_zz1000）
7. 保存为 `fundamental_aligned.parquet`（自动备份旧文件为 `.bak_YYYYMMDD_HHMMSS`）

输出字段: symbol, trade_date, open/high/low/close, volume, amount, adj_factor, raw_open/raw_high/raw_low/raw_close, ma5/ma10/ma20/ma60, rsi_6/rsi_14, kdj_k/kdj_d/kdj_j, macd_dif/dea/hist, vol_std_5/20/60, vol_atr_14, return_1d/3d/5d/10d/20d/60d, pct_change, bp, ep_ttm, beta_20, pe_ttm, pb, roe, total_mv, float_mv, turnover_rate, stock_name, industry, is_st, consecutive_limit_up_days, limit_up_today, limit_down_today, idx_hs300, idx_zz500, idx_zz1000 等

#### 3.4.2 拷贝到服务器

将生成的 parquet 文件从 Windows 机器拷贝到服务器项目目录:

```bash
# 从 Windows 拷贝到服务器
scp fundamental_aligned.parquet user@server:/path/to/project/db/custom/fundamental_aligned.parquet
```

#### 3.4.3 同步到数据库

**脚本**: `scripts/data/maintenance/sync_stock_daily_latest_from_parquet.py`

从 `db/custom/fundamental_aligned.parquet` 增量 upsert 到 PostgreSQL `stock_daily_latest` 表:

```bash
# 增量同步（只导入比数据库更新的交易日数据）
python scripts/data/maintenance/sync_stock_daily_latest_from_parquet.py

# 预览模式（不写入）
python scripts/data/maintenance/sync_stock_daily_latest_from_parquet.py --dry-run

# 自定义数据库连接
python scripts/data/maintenance/sync_stock_daily_latest_from_parquet.py --database-url "postgresql://quantmind:***@127.0.0.1:5432/quantmind"

# 指定回刷天数（默认回刷最近 30 天，覆盖修订数据）
python scripts/data/maintenance/sync_stock_daily_latest_from_parquet.py --refresh-days 7
```

也可使用兼容入口脚本:

```bash
python scripts/sync_to_postgres_v2.py
```

脚本流程:
1. 读取 parquet，对比数据库已有 max(trade_date) 找出新增数据
2. 按 refresh-days 回刷最近 N 天（覆盖通达信修订的近期数据）
3. 批量 upsert（INSERT ON CONFLICT UPDATE，每批 1000 行）
4. 同步完成后清除 Redis 数据状态缓存

#### 3.4.4 公司行为数据获取（除权除息）

**脚本**: `scripts/fetch_corporate_actions.py`

在通达信客户端机器上运行，获取全市场公司行为（除权除息）数据，用于复权因子校准和模拟盘权益价格计算。

```bash
# 拉取 2026 年 6 月数据
python fetch_corporate_actions.py --start 202606 --end 202606

# 拉取 2025-2026 两年全量
python fetch_corporate_actions.py --start 202501 --end 202612 \
    --output corporate_actions_2025_2026.csv

# 断点续传（跨月大范围推荐）
python fetch_corporate_actions.py --start 202501 --end 202612 \
    --output corporate_actions.csv --resume
```

运行环境: Windows，需通达信专业普及版 TQ 策略模块

输出字段: symbol, date, type, bonus（每股分红）, allot_price（配股价）, share_bonus（每股送股数）, allotment（每股配股比例）

## 4. 模型特征快照

### 4.1 说明

`db/feature_snapshots/` 是模型训练与推理用的数据集，按年份存储为 parquet 文件。每年约 600MB~1GB，包含 152~155 维特征，覆盖全市场约 5000+ 标的。

### 4.2 数据来源

特征数据由以下数据源计算生成：

| 数据源 | 说明 | 获取方式 |
|--------|------|----------|
| Level2 高频数据 | 逐笔成交、订单簿等微观结构特征 | 工作站计算 |
| 基础行情数据 | OHLCV + 技术指标 + 资金流 | fundamental_aligned.parquet |
| CSMAR 数据 | 复权因子、日个股回报率、指数行情等 | CSMAR 数据库 |

### 4.3 目录结构

```
db/feature_snapshots/
├── model_features_2016.parquet      # 2016年特征 (~603MB)
├── model_features_2016.metadata.json
├── model_features_2017.parquet      # 2017年特征 (~723MB)
├── model_features_2017.metadata.json
├── ...
├── model_features_2026.parquet      # 2026年特征
└── model_features_2026.metadata.json
```

### 4.4 关联数据目录

特征计算过程中产生的中间数据存放在以下目录：

| 目录 | 说明 |
|------|------|
| db/csmar/ | CSMAR 源数据 (复权因子表、后复权日行情、指数日行情、日个股回报率) |
| db/hf_features/ | Level2 高频特征中间文件 (按日期) |
| db/market_silver_v2/ | 银层数据 (基础特征) |
| db/feature_audit_v2/ | 特征审计文件 |
| db/feature_snapshots_v2/ | L1 特征中间文件 |

### 4.5 特征维度

每个年度 parquet 包含约 152~155 列，主要特征分类：

| 分类 | 前缀 | 示例字段 | 说明 |
|------|------|----------|------|
| 基础行情 | (无前缀) | open, high, low, close, volume, factor | OHLCV + 复权因子 |
| 动量特征 | mom_ | mom_ret_1d, mom_rsi_14, mom_macd_hist | 收益率/RSI/MACD/KDJ |
| 波动率特征 | vol_ | vol_std_20, vol_atr_14, vol_realized_rv | 已实现波动率/跳跃 |
| 流动性特征 | liq_ | liq_turnover_os, liq_amihud_20, liq_mfi_14 | 换手率/Amihud/MFI |
| 资金流特征 | flow_ | flow_net_amount, flow_vpin, flow_qsp | 净流入/VPIN/价差 |
| 风格因子 | style_ | style_bp, style_ep_ttm, style_beta_60 | BETA/估值/规模 |
| 行业特征 | ind_ | ind_ret_1d, ind_strength_20, ind_dispersion_20 | 行业动量/离散度 |
| 微观结构 | micro_ | micro_qsp_equal, micro_imbalance_volume | 有效价差/订单不平衡 |

### 4.6 生成与更新流程

特征快照由 3 个脚本分步计算，最终合并为 152 维年度宽表：

```
                    100维 (L1 本地)               52维 (L2 工作站)
                    ──────────────               ──────────────
CSMAR 数据 ──────→ compute_l1_incremental_new  │
银层数据 ────────→   .py (本地)                  │  calc_hf_52d_features.py (工作站)
                         │                       │  ← Level2 逐笔数据
                         ▼                       ▼
                    daily_l1_local_YYYYMMDD   hf_YYYYMMDD.parquet
                    .parquet                       │
                         │                          │
                         └────────┬─────────────────┘
                                  ▼
                    merge_l1_l2_into_yearly.py (本地)
                                  │
                                  ▼
                    model_features_YYYY.parquet (152维)
```

以下脚本来自企业版（quantmind），可按需复制使用。

#### 4.6.1 原始数据准备

##### CSMAR 数据

从 CSMAR 中转站（https://www.chaoslib.com/）手动下载，解压后放入 `db/csmar/` 目录。

152 维完整特征快照（`build_feature_snapshots.py`）共需 13 张 CSMAR 数据表：

**银层构建（build_silver_market.py）读取 2 张**:

| # | CSMAR 数据表 | 关键字段 | 用途 |
|---|-------------|----------|------|
| 1 | 日个股回报率文件 | Opnprc, Hiprc, Loprc, Clsprc, Dnshrtrd, Dnvaltrd, Dsmvtll, Dsmvosd | 不复权 OHLCV + 市值 |
| 2 | 股票价格复权因子表(日) | CumulateBwardFactor | 后复权因子 |

**build_feature_snapshots.py 直接读取 11 张**:

| # | CSMAR 数据表 | 关键字段 | 用途 |
|---|-------------|----------|------|
| 3 | 个股换手率表(日) | ToverOs, ToverTl | 流动性：换手率 |
| 4 | 日交易统计文件 | Toltrdtims, Tolstknva | 流动性：成交笔数/均量 |
| 5 | 个股买卖价差表(日) | Qsp_equal, Esp_equal, AQsp_equal, Num | 微观：价差 + 成交笔数回退 |
| 6 | 个股已实现指标表(日) | RV, RRV, RSkew, RKurt | 波动率：已实现波动率/偏度/峰度 |
| 7 | 个股跳跃指标表(日) | Z_Adj, RJV, SJV, ISJump | 波动率：跳跃指标 |
| 8 | 个股买卖不平衡指标表(日) | B_Amount, S_Amount, B_Num, S_Num, B/M/S 分级 | 资金流：净流入/订单不平衡 |
| 9 | 个股知情交易概率指标表(日) | VPIN | 资金流：知情交易概率 |
| 10 | 相对价值指标 | F100701A(PB), F100101B(PE_TTM) | 风格：估值因子 |
| 11 | 三因子模型指标(日) | RiskPremium1, SMB1, HML1 | 风格：三因子 |
| 12 | 国内指数日行情文件 | 指数收盘价 | 风格：beta 基准收益 |
| 13 | 公司文件 | Indcd, Nnindcd | 行业：行业代码 |

CSMAR 数据按日期子目录组织，例如 `db/csmar/20260524/`。每次下载后解压到新日期子目录。

##### Level2 逐笔数据

需要用户手动获取，放入 `baidudesk/` 目录。每天两个文件：

- `baidudesk/snapshot_YYYYMMDD.parquet` — Level2 快照数据（十档行情）
- `baidudesk/deal_YYYYMMDD.parquet` — Level2 逐笔成交数据

##### 银层数据构建

银层数据是 L1 特征计算的输入，基于 CSMAR 数据构建，存放在 `db/market_silver_v2/`。

**全量构建**（首次部署）:

**脚本**: `scripts/data/rebuild_v2/build_silver_market.py`

```bash
# 构建指定年份
python scripts/data/rebuild_v2/build_silver_market.py --years 2026
```

输入: `db/csmar/` 下的 CSMAR 日个股回报率文件 + 复权因子表

输出:
- `db/market_silver_v2/market_base_YYYY.parquet` — 不复权 OHLCV + factor + 换手率
- `db/market_silver_v2/adjustment_factors_daily_YYYY.parquet` — 复权因子
- `db/market_silver_v2/market_ohlcv_raw_YYYY.parquet` — 原始 OHLCV

**增量更新**（日常维护）:

**脚本**: `scripts/data/processing/incremental_update_silver.py`

```bash
# 从 CSMAR 新下载的 CSV 中读取缺失日期，追加到银层 parquet
python scripts/data/processing/incremental_update_silver.py
```

**补充换手率**:

**脚本**: `scripts/data/processing/add_turnover_to_silver.py`

```bash
# 从 CSMAR 后复权行情读取换手率，添加到银层
python scripts/data/processing/add_turnover_to_silver.py
```

#### 4.6.2 L1 本地特征计算 (100 维)

**脚本**: `scripts/data/processing/compute_l1_incremental_new.py`

在本地机器运行，从银层数据 + CSMAR 数据计算单日 L1 特征切片（约 100 维），包含动量、波动率、流动性、风格因子、行业特征等。

```bash
# 计算指定交易日的 L1 特征
python scripts/data/processing/compute_l1_incremental_new.py --date 2026-06-28

# 指定日期 + 输出审计文件
python scripts/data/processing/compute_l1_incremental_new.py --date 2026-06-28 --write-audit

# 自定义数据目录
python scripts/data/processing/compute_l1_incremental_new.py --date 2026-06-28 \
    --csmar-root db/csmar \
    --silver-dir db/market_silver_v2
```

输入:
- `db/market_silver_v2/market_base_YYYY.parquet` — 银层不复权 OHLCV + factor
- `db/csmar/` — CSMAR 复权因子表、日个股回报率、指数行情

输出:
- `db/feature_snapshots/daily_l1_new_YYYYMMDD.parquet` — 单日 L1 特征切片
- `db/feature_snapshots/daily_l1_new_YYYYMMDD.audit.json` — 审计文件（--write-audit 时）

L1 特征不含 flow_*/micro_*/vol_realized_* 等 L2 高频维度。

#### 4.6.3 L2 高频特征计算 (52 维)

**脚本**: `scripts/data/calc_hf_52d_features.py`

在远程工作站上运行，从 Level2 逐笔数据计算单日 L2 高频特征（约 52 维），包含资金流、微观结构、已实现波动率、跳跃等。

```bash
# 工作站上执行（单日）
python scripts/data/calc_hf_52d_features.py --date 2026-06-28

# 批处理模式（推荐）
python scripts/data/calc_hf_52d_features.py --date 2026-06-28 \
    --batch-size 500 --batch-workers 8
```

工作站信息:
- 地址: 8.149.9.248:58747 (root)
- 连接配置在 .env 中: WORKSTATION_HOST / WORKSTATION_PORT / WORKSTATION_USER / WORKSTATION_PASSWORD

输入:
- `baidudesk/snapshot_YYYYMMDD.parquet` — Level2 快照数据
- `baidudesk/deal_YYYYMMDD.parquet` — Level2 逐笔成交数据

输出:
- `db/hf_features/hf_YYYYMMDD.parquet` — 单日 L2 特征

> **注意**: --workers 参数在批处理模式下被忽略，实际并行度由 --batch-workers 控制（默认 8）。工作站 LXC 容器内存限制 29GB 时 8 workers 可能 OOM，需降到 4 或扩大内存。

#### 4.6.4 L1+L2 合并到年度宽表 (152 维)

**脚本**: `scripts/data/processing/merge_l1_l2_into_yearly.py`

将当日 L1 切片和 L2 切片按 (symbol, trade_date) 合并到年度 model_features 宽表。

```bash
# 合并指定日期
python scripts/data/processing/merge_l1_l2_into_yearly.py --date 2026-06-28

# 合并前备份年度宽表
python scripts/data/processing/merge_l1_l2_into_yearly.py --date 2026-06-28 --backup

# 过滤 B 股
python scripts/data/processing/merge_l1_l2_into_yearly.py --date 2026-06-28 --drop-b-shares
```

输入:
- `db/feature_snapshots/daily_l1_local_YYYYMMDD.parquet` — L1 切片
- `db/hf_features/hf_YYYYMMDD_*.parquet` — L2 切片
- `db/feature_snapshots/model_features_YYYY.parquet` — 年度宽表（已存在则追加，不存在则创建）

输出:
- `db/feature_snapshots/model_features_YYYY.parquet` — 更新后的 152 维年度宽表
- `db/feature_snapshots/model_features_YYYY.metadata.json` — 更新后的元数据

#### 4.6.5 全量重建 vs 增量更新

| 模式 | 脚本 | 适用时间 | 说明 |
|------|------|----------|------|
| 全量重建 | build_feature_snapshots.py | 周末 | 从银层 + CSMAR 全量重算 152 维，数据完整 |
| 增量更新 | run_daily_l1_l2_pipeline.py | 周一~周五 | 逐日追加，快速更新最新交易日 |

> **注意**: 增量更新脚本（分步模式）的 L2 高频数据覆盖率不足。如需完整的高频特征覆盖，建议使用全量重建脚本（4.6.6）。

#### 4.6.6 增量日更流水线（周一~周五）

**脚本**: `scripts/data/processing/run_daily_l1_l2_pipeline.py`

串联 L1 计算 → L2 计算 → L2 回传 → 合并年度宽表 全流程：

```bash
# 完整流程
python scripts/data/processing/run_daily_l1_l2_pipeline.py --date 2026-06-28

# 自定义工作站参数
python scripts/data/processing/run_daily_l1_l2_pipeline.py --date 2026-06-28 \
    --l2-batch-size 500 --l2-workers 8

# 跳过某些步骤
python scripts/data/processing/run_daily_l1_l2_pipeline.py --date 2026-06-28 \
    --skip-l2    # 跳过 L2 计算
```

流程:
1. 本地计算单日 L1 切片 (compute_l1_incremental_new.py)
2. SSH 到工作站计算单日 L2 切片 (calc_hf_52d_features.py)
3. 将 L2 切片同步回本地 (scp)
4. 合并 L1/L2 到年度 model_features 宽表 (merge_l1_l2_into_yearly.py)

> **注意**: 脚本不执行任何数据库写入。

#### 4.6.7 全量历史重建（周末）

**脚本**: `scripts/data/rebuild_v2/build_feature_snapshots.py`

从银层 + CSMAR 数据全量重建指定年份的 152 维特征快照，适合周末执行或历史数据修复。

```bash
# 重建指定年份
python scripts/data/rebuild_v2/build_feature_snapshots.py --years 2026

# 重建多年
python scripts/data/rebuild_v2/build_feature_snapshots.py --years 2024 2025 2026
```

输入:
- `db/market_silver_v2/market_base_YYYY.parquet` — 银层基础数据
- `db/csmar/` — CSMAR 源数据（13 张表）

输出:
- `db/feature_snapshots/model_features_YYYY.parquet` — 年度 152 维宽表
- `db/feature_snapshots/model_features_YYYY.metadata.json` — 元数据
