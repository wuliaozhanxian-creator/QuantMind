# 投研平台数据异常排查与修复报告 (2026-05-14)

## 1. 核心问题汇总

### 1.1 均线 (MA) 计算偏差
*   **症状**：数据库中的 `ma5` 和 `ma10` 与当前的 `close` 序列不匹配，导致技术分析指标失效。
*   **原因**：原始数据源在增量计算或导入过程中产生了逻辑漂移，或复权基准不统一。
*   **解决方案**：
    *   在 PostgreSQL 中使用窗口函数执行全量重算：
        ```sql
        UPDATE stock_daily_latest SET ma5 = calculated_ma5, ma10 = calculated_ma10 FROM (...)
        ```
    *   确保均线严格遵循最近 5/10 个交易日的收盘价数学平均值。

### 1.2 成交额 (Amount) 与成交量 (Volume) 的 100 倍量级误差
*   **症状**：部分个股（如 `SZ002580`）成交额显示为数十亿，而实际应为千万级；其他个股（如 `SH601607`）表现正常。
*   **原因**：原始数据在采集时单位不统一（分与元的混合），导致 100 倍的缩放误差。
*   **解决方案**：
    *   执行启发式校准：检测 `成交额 / 流通市值` 与 `换手率` 的偏离度。
    *   对偏离度超过 5 倍的异常行执行：`amount = amount / 100.0`, `volume = volume / 100.0`。

### 1.3 换手率 (Turnover Rate) 异常封顶 (1% Limit)
*   **症状**：投研平台列表中，几乎所有高活跃度标的的换手率均被锁定在 `1.00%` 或 `0.99%`，无法显示更高数值。
*   **原因**：原始数据源在导入时将 `turnover_rate` 截断在了 `1.0`（比例值）以下，且单位（比例 vs 百分比）极度混乱。
*   **解决方案**：
    *   **物理对齐**：不再信任原始数据源的换手率字段，利用已校准的成交额和市值进行重算：
        ```sql
        UPDATE stock_daily_latest SET turnover_rate = (amount / float_mv) * 100
        ```
    *   修正后，系统已能正常显示 `5%`、`10%` 等真实的高换手率。

### 1.4 K 线图价格二次复权误差
*   **症状**：K 线价格显示仅为实际股价的几分之一（如茅台显示为 150 元）。
*   **原因**：后端 `get_stock_kline` 接口在读取已处理的价格后，又冗余地除以了一次 `adj_factor`。
*   **解决方案**：移除接口层的除法逻辑，直接透传数据库中的除权价。

## 2. 后端适配逻辑 (research_service.py)

为确保前端展示的直观性与逻辑自洽，后端序列化层应用了以下规则：

1.  **MA 均线缩放**：由于数据库存储的是后复权均线（量级较大，如 11000），而前端显示的是当前除权价（量级较小，如 1300），后端在输出 `ma5/ma10` 时自动除以 `adj_factor`，确保均线与股价在 UI 上重合。
2.  **换手率透传**：后端不再对换手率进行 `*100` 操作，保持数据库中的百分比数值（如 1.48 代表 1.48%），由前端统一格式化。
3.  **单位换算**：成交额统一通过 `to_yi()` 函数转换为“亿元”单位，并保留 4 位小数。

## 3. 维护建议
*   **数据导入**：后续重新导入数据后，必须运行全量 MA 重算和换手率物理对齐 SQL。
*   **单位检查**：定期抽查 `成交额 / 流通市值` 的比例，确保量级在合理范围内。
